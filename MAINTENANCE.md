# Maintenance Guide

StartFit QR Evidence Hub는 발표 QR 보조 자료를 빠르게 수정하고 재배포할 수 있도록 정적 웹사이트 구조로 정리되어 있습니다.

## 파일 역할

| 파일 | 수정할 때 |
|---|---|
| `index.html` | 섹션 순서, 큰 HTML 구조, SEO 메타데이터를 바꿀 때 |
| `content.js` | 진단 과정 탭, 세부 화면 설명, 창업 유형 설명, 이미지 경로를 바꿀 때 |
| `icons.js` | 카드와 탭에 붙는 아이콘 종류나 매핑 규칙을 바꿀 때 |
| `script.js` | 메뉴, 탭, 진단 흐름도, 이미지 확대, 해시 링크 같은 상호작용을 바꿀 때 |
| `styles.css` | 색상, 여백, 반응형, 카드/탭 디자인을 바꿀 때 |
| `assets/` | 캡처 이미지나 검증 이미지를 교체할 때 |
| `.vercelignore` | Vercel 배포에서 제외할 로컬 임시 파일을 관리할 때 |
| `QR_TO_PRESENTATION_LINK_MAP.md` | 발표자료의 QR 연결 위치를 정리할 때 |

## 자주 하는 수정

### 진단 단계 문구 수정

`content.js`의 `diagnosisJourney` 배열에서 해당 큰 탭과 세부 단계의 `title`, `role`, `input`, `output`, `next`, `alt`를 수정합니다.

### 진단 단계 이미지 교체

1. 새 이미지를 `assets/live/diagnosis-journey/`에 넣습니다.
2. `content.js`에서 해당 단계의 `image` 경로를 새 파일명으로 바꿉니다.
3. 브라우저에서 큰 탭/세부 탭 이동, 이미지 깨짐, 확대 모달 여부를 확인합니다.

### 창업 유형 설명 수정

`content.js`의 `founderTypes` 배열에서 유형명, 핵심 질문, 의미, 우선 보완 액션을 수정합니다. 현재 live 예시 유형은 `featured: true`로 표시합니다.

### 카드 아이콘 교체

`icons.js`의 `textIconRules`, `navIcons`, `featureIcons`에서 텍스트와 아이콘 이름을 바꿉니다.

### 색상/톤 수정

`styles.css` 상단의 `:root` 색상 토큰을 먼저 바꿉니다. 개별 카드에 직접 색을 쓰기보다 토큰을 수정해야 전체 톤이 유지됩니다.

## 검증 체크리스트

- 데스크톱 첫 화면에서 제목, 대시보드 이미지, 모바일 프레임이 겹치지 않는다.
- 모바일 폭에서 가로 스크롤이 생기지 않는다.
- 메뉴 열기/닫기가 작동한다.
- 진단 과정 5개 큰 탭과 28개 화면이 모두 이동한다.
- 창업 유형 설명 5개 카드가 표시된다.
- 모든 주요 이미지는 미리보기에서 잘리지 않고, 더블클릭 또는 큰 화면 보기로 확대된다.
- 대시보드 탭과 세부 이미지 탭이 정상 전환된다.
- 모든 이미지 경로가 깨지지 않는다.
- 콘솔 오류가 없다.
- README와 실제 파일 구조가 일치한다.
